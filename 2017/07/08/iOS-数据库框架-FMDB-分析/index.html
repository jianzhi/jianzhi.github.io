主要的几个类：
  • FMDatabase:代表一个 SQLite 数据库实例，用于执行 SQL 语句。
  • FMResultSet:代表在FMDatabase中执行查询后的结果集。
  • FMDatabaseQueue:用于多线程读写数据库。

SQLite有三种线程模式：

  • 单线程(Single-thread)：所有互斥锁(mutex)都被禁用，多个线程同时访问 SQLite 是不安全的。
  • 多线程(Multi-thread)：一个数据库连接同一时间内只有一个线程使用就是安全的，多个线程同时使用同一个连接就是不安全的。
  • 串行(Serialized)： 开启所有锁，可以被多个线程任意访问。

在 iOS 中编译SQlite时默认使用的是多线程模式，sqlite3_threadsafe() 方法返回的是 2。如果返回 0 则代表是单线程模式，返回 1 代表是串行模式。

要想设置SQlite的线程模式，有很多种方式：

  • 编译期设定： 通过 SQLITE_THREADSAFE 这个参数进行编译器的设置来选择线程模型
  • 初始化设定： 通过调用 sqlite3_config() 可以在 SQLite 初始化时设置，例如想要设置串行模式：sqlite3_config(SQLITE_CONFIG_SINGLETHREAD);
  • 运行时设定： 通过调用 sqlite3_open_v2() 方法指定数据库连接的数据库线程模式
